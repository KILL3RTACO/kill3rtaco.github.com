ItemMail

Overview

ItemMail, now in version 3, has been far improved since the previous versions. New features include physical mailboxes, better enchantment handling and even detailed information for each mail/request sent!

ItemMail is designed to help trading. The idea originally came about about over a yea ago when players on a server I play on repeatedly commuted to a town/server spawn to trade items. With the introduction of auctions and player shops on this server, this has been greatly reduced. However, there still exists the possibility of friends or town/kingdom staff needing materials or equipment.

Definitions

ItemMail - Aside from being the name of the plugin, ItemMail refers to and/or represents 'mail' sent between players.

ItemRequest - Something that was introduced in early versions but not given a name. ItemRequests refer to and/or represent 'requests' sent from one player to another, where one player asks another for items.

NB! Before Updating

ItemMail will be using a new MySQL table to store data. It is extremely beneficial that you tell players on you server to open all wanted mail before installing Version 3.

Commands

/im mail-info [#] - See more detailed information about a specific ItemMail.

/im request-info [#] - See more detailed information about an ItemRequest.

/im mb [page] - Opens your ItemMailBox.

/im rb [page] - Opens your ItemRequestBox.

/im send >player< [item] [amount] - Sends ItemMail to a player.

/im request >player< >item< [amount] - Sends an ItemRequest to a player.

/im open [#] - Opens ItemMail.

/im delete [#/*] - Deletes ItemMail.

/im accept [#] - Accept an ItemRequest.

/im decline [#] - Decline an ItemRequest.

If -d is added at the end of the send command, the plugin will try to send to the player's physical mailbox instead. Try refers to the possibility that the player may not have one, or that it is full.
