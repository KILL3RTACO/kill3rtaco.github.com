About

When I saw that I was reusing code too many times in my plugins, I decided to make this. Most of which is inspired by DeityAPI by ImDeity/vanZeben. I wouldnt say that I copied their code, but maybe copied quite a bit of what I liked and tweaked it to make it easier for me.

Differences Between DeityAPI and TacoAPI

The two plugins are similar in many ways. DeityAPI I admit is far superior. But a lot of the custom plugins they use for the server use WorldGaurd, WorlEdit and various other plugins. Mine don't (with the exception of Mineopoly, to paste the board), so I just used some of the basic utilities in DeityAPI.

These 'utilities' mostly includes the command handling.

DeityCommandHandler and TacoCommandHandler

The similarities between these are almost identical. I added an alias system the command handling, something the DeityCommandHandler doesn't have (since I last checked). This means that a 'subcommand' (the second "word" when typing a command) can have different names.

For example, in ItemMail:

/im send kill3rtaco diamond 64

Is the same as

/im gift kill3rtaco diamond 64

DeityCommandReceiver and TacoCommand

These two classes are used to run the command. I removed the 'Receiver' part of the name (didn't need it), so obviously thats a difference. TacoCommands, as mentioned earlier, can have aliases
